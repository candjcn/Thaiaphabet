泰语字母渐进式互动学习网页 (V2.0 需求文档)

1. 项目概述

定位：面向泰语零基础初学者的游戏化、关卡式、即学即用的字母与拼读学习网页。

核心痛点解决：摒弃传统“先背完 44 个辅音、32 个元音再学拼读”的枯燥模式。采用“玛尼与马纳（Manee Mana）”式的渐进拼读教学法。每课只学 3-4 个字母，学完立刻拼出实用单词和句子。

技术特色：纯视觉画布临摹（Canvas）、自适应艾宾浩斯复习算法（SRS）、网页音频系统（Web Audio）与即时游戏化反馈。

2. 课程体系设计 (Pedagogical Strategy & Lesson Plan)

我们将泰语辅音、元音打散，编排进 10 个递进课时。难度从简单到困难，让用户体验“无痛通关”。

3. 核心功能与交互设计 (Core Features)

3.1 单课通关流程 (Lesson Learning Loop)

每个课时（Lesson）均由 4 个微小步骤（Steps）组成，完成一个步骤耗时不超过 2 分钟，确保轻量高频：

[步骤1: 字母探索 (Learn)] -> [步骤2: 笔顺临摹 (Trace)] -> [步骤3: 单词&句子合成 (Synthesize)] -> [步骤4: 闯关测试 (Quiz)]


步骤1: 字母探索 (Explore)

展示本课的 3 个字母。

点击字母，大声播放辅音名字（如 “Maw-Ma”），并显示其对应的精美卡片（马的插画）。

步骤2: 纯视觉手写临摹 (Visual Tracing Canvas)

底图辅助：中央显示半透明的泰语字母 SVG 骨架。

笔顺动画引导：页面上有一个“播放笔顺”按钮，点击后，一个发光的小球会在字母骨架上按正确笔顺（泰语写字必须先画圆圈）滑动。

自由描红区：提供一个 Canvas 画布，用户可以直接用鼠标/手指在半透明底图上涂鸦、临摹。

辅助按键：【清除重写】、【显示/隐藏笔顺指南】。

步骤3: 单词与句子合成 (Word & Sentence Synthesis)

拼读动画：展示字母相加的过程。例如：ต [To] + า [a] = ตา [Ta]，点击字母时分别发音，拼合在一起时播放完整单词音。

句子拆解：展示句子 “ตา ดู ปู”（外公看螃蟹）。点击其中任何一个单词都会高亮并播放声音，帮助用户建立单词与句子关联。

步骤4: 关卡小测试 (Gamified Quiz)

听音选字：播放 “Po”，用户从 3 个字母中选出正确的。

句子组装拼图：提供打乱的泰语单词卡片（[ตา] [ดู] [ปู]），用户拖拽卡片拼成中文提示的句子（“外公看螃蟹”）。

3.2 艾宾浩斯复习系统 (Ebbinghaus SRS Engine)

为了防止用户“边学边忘”，设计“每日复习引擎 (Review Engine)”。

记忆状态跟踪：后台使用类似 SuperMemo-2 算法简化版。每个字母、单词都有一个“记忆强度值 (Memory Strength, 0% ~ 100%)”和“下次复习时间戳 (Next Review Timestamp)”。

复习波段：基于艾宾浩斯曲线，复习间隔设置为：30分钟、12小时、1天、2天、4天、7天、15天。

“复习能量罐” (Review Dashboard)：

用户每天登录网页，首页会显眼地提示：“你有 5 个字母/单词需要温习！”。

完成复习（通常是一个 1 分钟的快速听音选择或拼写）后，这些词汇的记忆强度恢复至 100%，复习能量罐充能完毕，用户才可以解锁新的课程。

实时时间监测 (Timer)：

每次开始一个 Lesson 均启动计时器。

通关后展示：“本次学习用时：01:45，专注度极高！”。

星级奖励机制 (Star Rating & XP)：

根据步骤 4 闯关测试的正确率与用时，给予 1 到 3 颗星 的评级。

奖励 XP（经验值），并在界面上用进度条展示用户的等级提升。

炫酷的视觉反馈：

当用户写完字母，或完成测试时，页面弹出金色向日葵/小星星。

通关时，网页触发虚拟五彩纸屑（Confetti 动画效果）和叮咚提示音。

学习卡片收藏册 (Collection Book)：

所有通关学过的字母、单词，都会收录在用户的“藏书阁”中，变成闪闪发光的“卡牌”，用户随时可以点击它们听发音，成就感直观可视。

4. 技术与架构设计 (Technical Architecture)

前端框架：HTML5 + Tailwind CSS (保证美观响应式) + Native JavaScript / Vue.js。

音效与音频：使用 HTML5 <audio> 标签。

建议预加载（Preload）常用音效（通关音、错误音、点击音）。

手写画布：使用 HTML5 <canvas>，捕获 mousedown / mousemove（PC端）和 touchstart / touchmove（移动端）实现流畅划线。

数据本地存储 (Local Storage)：

用户的学习进度（Unlock Lessons）、艾宾浩斯记忆数据（SRS Items）、总 XP、每日打卡天数（Streak），全部保存在浏览器的 localStorage 中。零基础建站无需配置复杂的后端数据库，即可实现断点续学。

5. 后续优化迭代考量 (Future Roadmap)

我们在后续迭代中，还可以围绕这个 PRD 讨论加减以下功能：

【加】泰语声调计算器 (Tone Calculator Tool)：泰语有五个声调，拼读规则复杂。未来可加入一个交互工具，用户输入“辅音+元音+声调”，网页自动计算并朗读正确声调。

【加】联机 PK/排行榜：允许用户与在线的其他初学者进行 60 秒听音辨字 PK，增强社交粘性。

【减】句子拼图：如果前期开发周期紧张，步骤 4 的句子组装可简化为多选题，二期再做拖拽式拼图。
